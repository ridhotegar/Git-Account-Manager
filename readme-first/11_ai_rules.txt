General Rules

Never rewrite the entire project.

Modify only affected files.

Preserve architecture.

Preserve naming consistency.

Never introduce unnecessary dependencies.

Always explain architectural decisions.

When implementing a feature:

1. Analyze existing code.

2. Create implementation plan.

3. Implement backend.

4. Implement frontend.

5. Write tests.

6. Update documentation.

Never skip tests.

Never create placeholder code unless explicitly requested.

Every feature must be production ready.

Prefer incremental commits.

If requirements are ambiguous, ask before implementing.
